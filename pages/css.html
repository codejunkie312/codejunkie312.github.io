    Concept(specificity) specificity is in this order in this order !important>id>class>tag
    :link is a css pseudo-class state to define a styling for unvisited clicked links
    :visited is a css pseudo-class state to define a styling for already clicked links
    :hover is a css pseudo-class state if added in front of a selector will create a different effect depending on it's content from the selector without it, if hover over the element on page
    :active is a css pseudo-class state if added in front of a selector will create a different effect for clicking on the element and pseudo-class and the three before it should be used in the order they are mentioned in this table
    transition a property to be used in pseudo-class selectors like the one mentioned before this example: "transition: margin-right 2s;", "transition: margin-right 2s .5s;", "transition: margin-right 2s ease-in-out;", "transition: margin-right 2s ease-in-out .5s;", "transition: margin-right 2s, color 1s;", "transition: all 1s ease-out;" Can also be used in none pseudo selector version to ease off the effect transition
    cursor a property to define how the cursor looks when hovering the selector element and it can the following values: "pointer", "cursor", "default"
    margin Creates space between elements if there are any near container otherwise doesn't do anything
    padding is a property and is the area right after content box and creates spacing between containers because it increase the size of the box if max-width/height is not defined in the parent container
    font-family a property to define fonts to use and can take in multiple values and respects the order of the fonts mentioned, and it is good practice to input fallback fonts in case the user doesn't have the font installed in their machine with "serif" and "sans-serif" being the final fallback fonts
    font-size a property to define the size of text that will be inputed within the selector and if not inputed will revert to the default value or it will inherit from teh parent containers
    font-weight a property to define text thickness, ether a value can be inputed or a keyword with normal = 400 and cold = 700 and value is betwen 0 to 1000 and it is good practice to input value in increments of 100
    text-decoration a property to define text decoration for the selector elements takes multiple values seperated by space from among (underline, none, inherit, unset) and (dotted, solid) plus a color at the end (optional) this can be inputed in one single line or seperated between text-decoration-line, text-decoration-color, text-decoration-style, text-decoration-thickness
    background-color a property to define the background color of a container the values can ether be keyword colors which are 144 or a hex value #FFFFFF with each pair of number defining RGB values and also it can take rgb(0,0,0) with each value going from 0 to 255 or hsl which stands for hue, saturation and lighting with first value being a value between 0 and 360 and second and third value being percentages hsl(320, 50%, 90%) and all of these values can take a forth element which is called an alpha element which defines the opacity of the color and it is a number that starts from 0 to 1 rgba(a,b,c,0.8) or hsla(a,b,c,0.5) or to hex value by adding a forth pair
    border a property to define the border and it take three values and in this order styling (dotted, solid,...etc) and thickness in the form of a number (2px) and color of the border
    border-radius a property to define the shape of the corners of the border and it can take a value or a percentage with 50% being a perfect circle and 100% an oval shape
    box-shadow a property to define the shadow behind the container, and can take mutilple methodes of inputs read on it in mozilla, but the most used is pixels for each of the four corners of the box in clock wise order starting from top and lastly a color
    float this is a property that enables the content to float around the containing box depending on the value it recieves.
    ::before a pseudo-class that is used usually with the content property to add logos before the content of the element and ::after acts the same way and is used to add logos after content
    content is a property that add content ether before or after in the content section of the element using the pseudo-classes ::before and ::after and this content can be anything(text,pic,....etc)
    vertical-align is a property that can be used vertically position the element selected click to see the values it can take and how they display
    display: flex; flex is a display value that utilizes a column grid system to position items within a page and it breaks up to flex container and within it flex items and these two take different properties to be deisgned please visit link to know more
    justify-content a property usually used in flex-items in order to define how they position inside their flex-container on the main axis while it shrinks and expands depending on user screen size
    align-items is a property typically used with flex-items in order to alter their vertical positioning and it takes multiple values please read up in the link provide to understand the behaviour of each one
    flex-grow is a property that is used on flex-items in order to define how much they grow as the screen shrinks and expands and it takes unitless values (default value is 0)
    flex-shrink is a property that is used on flex-items in order to define how much they shrink as the screen shrinks and it takes unitless values (default value is 1)
    flex-basis is a property that is used on flex-items in order to define the width the item is shrinking from
    flex is a property that is used on flex-items and it is used to define the values for flex-grow/flex-shrink/flex-basis all in one line
    flex-wrap is a property that is used on flex-containers and defines items react inside the container when they can't fit within it and it can take three values: wrap, wrap-reverse, nowrap(default)
    align-content is a property that is used on flex-containers in order to define how the items group when flex-wrap is used
    flex-direction is a property that is used on flex-containers in order to define how thy position from left to right or right to left or top to bottom or the other way around no matter what their order in the html code see values it can take by going to mdm link
    flex-flow is a property that is used in order to define both flex-direction and flex-wrap in that order in one line
